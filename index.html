COMPETITION STRUCTURE

HACKATHON PHASES

Step-by-step journey from problem understanding to final pitch

PHASE 1

PROBLEM UNDERSTANDING & IDEATION

Duration: 2-3 Hours

DELIVERABLE

1-page Solution Brief covering: Problem understanding, Target user, AI opportunity, Agent roles (who does what)

CHECKPOINT REVIEW

Mentors/Judges review. Teams not meeting AI + Agent clarity will be redirected.

PHASE 2

AGENT ARCHITECTURE DESIGN

Duration: 3-4 Hours

MANDATORY OUTPUT

Agent Architecture Diagram showing: Agents involved, Tools used by each agent, Memory type (short-term/vector DB), Communication flow

MUST ANSWER

Why agents, not a single model? How agents collaborate? What decisions are automated?

PHASE 3

DATASET & MODEL STRATEGY

Duration: 3 Hours

DELIVERABLE

Dataset source (real/synthetic/scraped), Preprocessing approach, Model selection (LLM: OpenAI/Llama/Gemini/Mistral or Classical ML/Deep Learning), Prompt strategy

BONUS POINTS

RAG is used, Fine-tuning logic is explained (even if not executed)

PHASE 4

PROTOTYPE DEVELOPMENT

Duration: 10-16 Hours

MINIMUM WORKING PRODUCT

Frontend (Streamlit/React/Gradio), Backend (FastAPI/Flask), Agent orchestration (LangGraph/CrewAI/AutoGen/Custom logic), At least one end-to-end demo flow

IMPORTANT RULE

Hard-coded demos = ‚ùå Must show dynamic agent decision-making

PHASE 5

VALIDATION & METRICS

Duration: 2 Hours

TEAMS MUST SHOW

Accuracy/relevance metrics, Latency or response quality, Failure case handling. Even qualitative metrics are acceptable if justified.

PHASE 6

PITCH & DEMO (FINAL ROUND)

Duration: 6-8 Minutes

PITCH STRUCTURE

Problem & pain point (1 min), AI agent architecture (2 mins), Live demo (2-3 mins), Impact, scalability, future scope (1-2 mins)
